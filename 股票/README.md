# Stock Ideas Dashboard

Stock Ideas Dashboard is a Next.js 14 MVP for stock idea discovery and scoring. It uses mock market data today, but the data access layer is isolated behind a provider interface so it can later be replaced with Yahoo Finance, Alpha Vantage, Polygon, or an internal data service.

## Features

- `/ideas` dashboard with trading-terminal dark UI
- Stock idea cards plus sortable table
- Fixed filter bar for sector, score range, signal, and sort mode
- `/stock/[ticker]` detail page with fundamentals, K-line chart, volume chart, trend metrics, and score breakdown
- TypeScript scoring engine with 0-100 score and BUY/HOLD/SELL signal
- API routes:
  - `GET /api/ideas`
  - `GET /api/stock/[ticker]`
- Mock universe of 20 stocks with 7-day and 30-day price series

## Scoring Formula

```ts
score =
  price_momentum_7d * 0.3 +
  volume_change * 0.2 +
  moving_average_trend * 0.3 +
  volatility_score * 0.2
```

Ideas are generated by `generateIdeas(stocks)`:

- daily change must be greater than 2%
- total score must be greater than 60
- results are sorted by score descending

## Architecture

```text
app/                  Next.js App Router pages and API routes
components/           Reusable dashboard, table, card, chart, badge components
data/                 Deterministic mock market data generator
lib/                  Scoring and market data provider layer
types/                Shared TypeScript domain types
utils/                Formatting and numeric helpers
```

The provider contract lives in `lib/marketData.ts`. To connect real data, implement `MarketDataProvider` and return it from `getMarketDataProvider()`.

## Run Locally

```bash
npm install
npm run dev
```

Open the local URL printed by Next.js and visit `/ideas`.

## Environment

Create `.env.local` from `.env.example`:

```bash
cp .env.example .env.local
```

`STOCK_API_KEY` is reserved for a future real market data provider. `NEXT_PUBLIC_APP_URL` should be set to your deployed Vercel URL in production.

## Deploy to Vercel

1. Push the project to GitHub, GitLab, or Bitbucket.
2. Create a new Vercel project and import the repository.
3. Set environment variables in Vercel:
   - `STOCK_API_KEY`
   - `NEXT_PUBLIC_APP_URL`
4. Keep the framework preset as Next.js. `vercel.json` already defines the build, install, and dev commands.
5. Deploy. The production app entry point is `/ideas`.
